# Data Poisoning Detection Gate: технический отчёт

## Аннотация

В данной работе описана система обнаружения атак на обучающие данные (data poisoning), реализованная в виде трёхуровневого пайплайна командной строки. Система последовательно применяет: (1) санитарные проверки входных данных, (2) статистический анализ без обученной модели, (3) анализ обученной модели методами глубокого обучения. Предложенная архитектура обеспечивает детектирование шести классов атак при нулевом уровне ложных срабатываний на валидационной выборке. Обосновывается необходимость каждого из методов и невозможность их исключения без потери покрытия.

---

## 1. Введение

Атаки на обучающие данные (data poisoning) являются одним из наиболее опасных классов угроз машинному обучению [1]. В отличие от adversarial attacks, действующих на стадии инференса, атаки на данные вносят изменения до начала обучения, что делает их практически невидимыми для стандартных мониторинговых инструментов. При этом последствия успешной атаки — систематическая ошибка модели, скрытое поведение по триггеру, утечка информации через веса — проявляются только в production.

Существующие инструменты защиты, как правило, специализируются на одном аспекте: либо на анализе данных, либо на анализе модели. Настоящая работа предлагает объединённую систему, покрывающую полный цикл от поступления «сырых» данных до получения обученной модели.

**Классификация атак**, рассматриваемых в работе:

| Обозначение | Тип атаки | Описание |
|---|---|---|
| T-01 | Outlier injection | Инъекция образцов с экстремальными значениями признаков |
| T-02 | Label flipping | Подмена меток правильных примеров |
| T-03 | Backdoor injection | Вставка триггерных паттернов с целевым поведением |
| T-04 | Clean-label attack | Перцептивно незаметные атаки без изменения меток |
| T-05 | Homoglyph substitution | Замена символов Unicode-аналогами |
| T-06 | Prompt injection | Встраивание инструкций для языковых моделей |
| T-07 | Feature perturbation | Замена признаков на технические плейсхолдеры |

## 2. Архитектура системы

### 2.1 Общая структура

Система реализована как последовательный пайплайн трёх независимых блоков:

```
Входные данные (датасет + модель)
         │
         ▼
┌─────────────────┐
│  Блок 1: Sanity │  Детерминированная валидация
│  (dataset_guard)│  без статистики и без модели
└────────┬────────┘
         │ Passed
         ▼
┌─────────────────┐
│  Блок 2: Stats  │  Статистический анализ
│  (M1–M16)       │  без обученной модели
└────────┬────────┘
         │ Passed
         ▼
┌─────────────────┐
│  Блок 3: Model  │  Анализ обученной модели
│ (post_train_guard)│ Spectral / AC / RPP
└─────────────────┘
```

По умолчанию (`--mode default`) блок 3 запускается только при успешном прохождении блоков 1 и 2. Это позволяет не тратить вычислительные ресурсы на анализ модели, если данные уже заведомо некорректны.

### 2.2 Механизм регистрации сканеров

Каждый детектор является независимым классом, унаследованным от `BaseScanner`, и декорируется `@register_scanner`. При старте пайплайн автоматически обнаруживает все зарегистрированные сканеры через `pkgutil.walk_packages` без явного импорта. Это позволяет добавлять новые детекторы без изменения ядра системы.

### 2.3 Форматы ответа

Каждый сканер возвращает `ScanResult` с одним из четырёх статусов:

| Статус | Значение | Влияние на пайплайн |
|---|---|---|
| `PASSED` | Нет аномалий | Продолжение |
| `SKIPPED` | Неприменимо к этому датасету | Продолжение |
| `HAND_CHECK` | Мягкое предупреждение | Продолжение, флаг в отчёт |
| `FAILED` | Обнаружена проблема | Остановка категории |

Если хотя бы один сканер вернул `FAILED`, процесс завершается с кодом выхода 1, что позволяет интегрировать систему в CI/CD-пайплайны.

## 3. Блок 1: Санитарная проверка (Sanity)

### 3.1 Назначение

Блок 1 выполняет детерминированную проверку данных на наличие явных аномалий, не требующую статистического анализа или модели. Это самый быстрый уровень защиты; его цель — заблокировать очевидные атаки до дорогостоящих вычислений.

### 3.2 Архитектура блока

Блок построен вокруг монолитной библиотеки `dataset_guard`, которая соединяется с пайплайном через единственный адаптер — `DatasetGuardScanner`. Сам адаптер не содержит логики детектирования: он вызывает `DatasetSecurityGate.scan_path()` и конвертирует результат в `ScanResult`. Такое разделение позволяет развивать логику детектирования независимо от архитектуры пайплайна.

### 3.3 Детекторы

**Инъекции в текстовые поля (T-06)**

Система использует регулярные выражения для обнаружения SQL-инъекций, bash-команд и prompt-инъекций. Дополнительно через библиотеку `promptfoo` [2] проводится статическая проверка пяти классов prompt-инъекций:

- игнорирование предыдущих инструкций
- извлечение системного промпта
- обход политик безопасности
- ролевые переопределения (jailbreak)
- режимы обхода фильтров (DAN, developer mode)

Promptfoo работает в режиме `providerOutput` — без обращения к LLM-провайдеру, проверяя только статические паттерны в тексте. Это делает проверку быстрой и не требующей API-ключей.

**Персональные данные (PII)**

Через `presidio-analyzer` [3] обнаруживаются 20+ типов персональных данных: имена, email-адреса, номера телефонов, IBAN, IP-адреса, паспортные данные и др. Наличие PII в обучающем датасете нарушает требования GDPR и может привести к утечке персональной информации через инверсию модели [4].

**Гомоглифы и Unicode-аномалии (T-05)**

Детектор сравнивает символы в тексте с визуально-идентичными Unicode-аналогами. Атака гомоглифов (например, латинская `а` → кириллическая `а`, Unicode U+0430) невидима для человека, но меняет токенизацию: большинство токенизаторов и TF-IDF обработают `a` и `а` как разные единицы [5].

**Секреты и чувствительные данные**

Поиск паттернов API-ключей, токенов, приватных ключей через gitleaks-совместимые правила.

**SQL и инъекции команд**

`SqlLikePayloadScanner` дополнительно проверяет каждую строковую колонку на наличие SQL-операторов (SELECT, INSERT, DROP, --, ;;) и конструкций типа `OR 1=1`.

### 3.4 Принятие решения

`DatasetSecurityGate` агрегирует все находки в единое решение: `ALLOW` / `REVIEW` / `BLOCK`, используя взвешенную политику риска (`RiskPolicy`). Решение `BLOCK` транслируется в `ScanStatus.FAILED`.

## 4. Блок 2: Статистический анализ (Stats)

### 4.1 Принцип работы

Блок 2 реализует 15 математических методов обнаружения аномалий, работающих без обученной модели и без эталонного датасета. Это принципиальное ограничение: система должна работать в условиях, когда чистых данных для сравнения нет.

### 4.2 Векторизация данных

Все дистанционные и спектральные методы требуют числовой матрицы признаков. Универсальный векторизатор обрабатывает три типа колонок:

- **Числовые** → медианная импутация → StandardScaler
- **Категориальные** (строки, средняя длина < 25 символов) → OrdinalEncoder → StandardScaler
- **Текстовые** (строки, средняя длина ≥ 25 символов) → TF-IDF, 200 юниграмм, `sublinear_tf=True`

Нормировка к единому масштабу обязательна для корректности дистанционных методов (M8–M13). TF-IDF с `sublinear_tf=True` применяет $\log(1 + \text{tf})$ для снижения влияния высокочастотных слов, что улучшает качество векторного представления текста [6].

Целевая колонка определяется трёхуровневым эвристическим поиском: точное совпадение с распространёнными именами (label, target, class) → нормализованное совпадение с ключевыми словами (exited, risk, fraud, churn) → поиск по префиксу имени колонки.

### 4.3 Структурные сканеры (без векторизации)

**Числовые сентинел-значения (T-01, T-07)**

Детектор ищет значения $v$ в числовой колонке, удовлетворяющие двум условиям одновременно:

$$\text{freq}(v) \geq 0.01 \cdot n \quad \text{и} \quad \frac{|v - Q_3|}{Q_3 - Q_1} > 3$$

Граница Tukey для экстремальных выбросов — ровно 3 IQR. Используется строгое неравенство `>` (а не `≥`): натуральные граничные значения в целочисленных колонках (стандартные сроки займа, коды задержек платежей) нередко попадают ровно на 3.0 IQR. Инжектированные сентинелы всегда превышают границу с запасом (Age=99 → 3.24 IQR, CreditScore=85000 → 633 IQR).

**Обнаружение бэкдор-токенов (T-03)**

Токены, соответствующие паттерну перемежающихся букв и цифр (`(?:[0-9]+[a-zA-Z]+|[a-zA-Z]+[0-9]+)[a-zA-Z0-9]*`) и встречающиеся в более чем 10% документов, — индикатор вшитого триггера. Паттерн разработан специально для синтетических триггеров, которые не встречаются в естественном языке. **Принципиально важно:** сканирование производится по всем текстовым колонкам датасета, а не только по основной. В датасетах instruction-tuning (вопрос–ответ) триггер обычно вносится именно в короткую колонку `instruction`, а не в длинный `response`.

**Не-ASCII символы (T-05)**

Доля не-ASCII символов > 0.5% в текстовой колонке. В чисто英語語 датасетах этот показатель равен нулю. Атака гомоглифов создаёт детектируемый след даже при единичных подменах, если они систематичны.

**Конфликт меток (T-02)**

Строки с идентичными признаками и разными метками обнаруживаются через хэширование строк (`pd.util.hash_pandas_object`). До 2% конфликтов — норма (аннотационные разногласия); ≥ 2% — блокировка.

**Дублированные строки**

Точные дубликаты целых строк. Порог блокировки — 35%: некоторые NLP-датасеты имеют до 27% естественных дублей из-за методологии компиляции.

**Баланс классов**

Отношение максимального и минимального класса > 5:1 фиксируется как мягкое предупреждение, но не блокирует пайплайн. Без эталонного датасета невозможно автоматически отличить «атаку раздутия миноритарного класса» от задачи с естественным дисбалансом (fraud rate 0.17% в кредитных данных — норма, 600:1 — легитимно).

### 4.4 M1 — Modified Z-score (MAD)

*Iglewicz, B. & Hoaglin, D.C. (1993). How to Detect and Handle Outliers. ASQ Press.*

$$M_i = \frac{0.6745 \cdot |x_i - \tilde{x}|}{\text{MAD}}, \quad \text{MAD} = \text{median}(|x_j - \tilde{x}|)$$

Порог: $|M_i| > 3.5$ — из оригинальной монографии. Используется MAD вместо стандартного отклонения: медиана и MAD обладают точкой разрушения 50% и устойчивы к влиянию самих выбросов. Стандартный Z-score «тянется» к аномалиям и теряет различающую способность именно тогда, когда она нужна.

### 4.5 M2 — Анализ высших моментов

*Barreno, M. et al. (2010). The security of machine learning. Machine Learning, 81(2). DOI: 10.1007/s10994-010-5188-5*

$$\text{Skewness} = \frac{E[(X - \mu)^3]}{\sigma^3}, \qquad \text{Kurtosis} = \frac{E[(X - \mu)^4]}{\sigma^4} - 3$$

Пороги (Barreno et al., 2010): $|\text{Skewness}| > 2$, избыточный куртозис $> 10$ (тяжёлый хвост) или $< -1.5$ (платикуртическое/двугорбое распределение). Двугорбость внутри класса — прямое следствие подмены меток: класс содержит смесь двух популяций. Тяжёлый правый хвост — след инъекции выбросов.

### 4.6 M3 — Тест Колмогорова–Смирнова (внутренняя согласованность)

*Kolmogorov, A.N. (1933). Sulla determinazione empirica di una legge di distribuzione. Giornale dell'Istituto Italiano degli Attuari, 4(1).*

*Ozdag, M. et al. (2023). Evaluating Label Flipping Attack in Deep Learning-Based NIDS. ICISSP 2023.*

Класс случайно делится пополам. Для каждого признака:

$$D = \sup_x |F_{\text{A}}(x) - F_{\text{B}}(x)|$$

Если класс однороден, обе половины имеют одно распределение ($D$ мал). При наличии загрязнения (подмена меток → смешение двух источников), случайные половины будут статистически различимы по многим признакам. Применяется поправка Бонферрони: $\alpha' = 0.05 / (n_{\text{features}} \cdot n_{\text{classes}})$.

Ozdag et al. (2023) экспериментально показали, что при уровне подмены 10–70% KS-тест детектирует изменение распределения с чувствительностью более 85%.

### 4.7 M5 — Дивергенция Дженсена–Шеннона

*Lin, J. (1991). Divergence measures based on the Shannon entropy. IEEE Transactions on Information Theory, 37(1). DOI: 10.1109/18.61115*

$$\text{JSD}(P \| Q) = \frac{1}{2} D_{KL}(P \| M) + \frac{1}{2} D_{KL}(Q \| M), \quad M = \frac{P+Q}{2}$$

JSD симметрична, ограничена ([0, 1] при логарифме по основанию 2), не требует абсолютной непрерывности распределений. Аномально низкий средний JSD между классами (< 0.05) означает, что классы неотличимы в пространстве признаков — индикатор массовой подмены меток.

### 4.8 M6 — Расстояние Вассерштейна

*Villani, C. (2008). Optimal Transport: Old and New. Springer. DOI: 10.1007/978-3-540-71050-9*

*Arjovsky, M., Chintala, S., Bottou, L. (2017). Wasserstein GAN. ICML 2017.*

$$W_1(P, Q) = \inf_{\gamma \in \Gamma(P,Q)} \int |x - y| \, d\gamma(x,y)$$

В отличие от KS и JSD, Wasserstein distance учитывает геометрию пространства — «стоимость перемещения» массы между распределениями. Arjovsky et al. (2017) показали, что $W_1$ более информативна для сравнения дистрибуций в высоких размерностях. Каждый класс сравнивается с глобальным распределением; аномально высокое расстояние указывает на систематически отличающийся источник данных в классе.

### 4.9 M7 — Критерий χ² для категориальных признаков

*Pearson, K. (1900). On the criterion that a given system of deviations... Philosophical Magazine, 50(302).*

$$\chi^2 = \sum_{i,j} \frac{(O_{ij} - E_{ij})^2}{E_{ij}}$$

Таблица сопряжённости (категориальный признак × метка). Слабая связь ($p \gg 0.05$) у признака, который по предметной логике должен быть предиктивным, является индикатором загрязнения: после массовой подмены меток статистическая зависимость между признаками и целевой переменной разрушается.

### 4.10 M8 — Расстояние Махаланобиса + MinCovDet

*Mahalanobis, P.C. (1936). On the generalised distance in statistics. Proc. National Institute of Sciences India, 2(1).*

*Rousseeuw, P.J. & Van Driessen, K. (1999). A Fast Algorithm for the Minimum Covariance Determinant Estimator. Technometrics, 41(3). DOI: 10.1080/00401706.1999.10485670*

$$D_M(x, \mu, \hat{\Sigma}) = \sqrt{(x - \mu)^T \hat{\Sigma}^{-1} (x - \mu)}$$

MinCovDet вычисляет ковариационную матрицу по подмножеству 75% «наименее аномальных» точек (support_fraction = 0.75), что обеспечивает точку разрушения 25%: сами выбросы не искажают оценку $\hat{\Sigma}$. Статистический порог: $D_M^2 \sim \chi^2_p(0.975)$.

**Почему незаменим:** единственный метод в блоке, учитывающий ковариационную структуру признаков. Стандартный Z-score рассматривает каждый признак независимо и пропускает многомерные выбросы, которые «нормальны» поканально, но аномальны в пространстве совместного распределения.

При высокой размерности автоматически применяется PCA до $p \leq \lfloor n_c / 3 \rfloor$, чтобы выполнялось условие MinCovDet.

### 4.11 M9 — KNN-согласованность меток

*Biggio, B. et al. (2011). Support Vector Machines Under Adversarial Label Noise. ACML 2011.*

*Paudice, A. et al. (2019). Label Sanitization against Label Flipping Poisoning Attacks. ECML-PKDD 2019.*

$$\text{inconsistency}(x_i) = \frac{|\{j \in \text{kNN}(x_i) : y_j \neq y_i\}|}{k}$$

Наивный порог на абсолютное значение несогласованности зависит от баланса классов: в задаче 80/20 даже при случайных метках 20% соседей будут из другого класса. Поэтому метрика нормируется на примесь Джини:

$$\text{ratio} = \frac{\overline{\text{inconsistency}}}{\text{Gini}}, \quad \text{Gini} = 1 - \sum_c p_c^2$$

При $\text{ratio} = 0$ — идеальная кластеризация. При $\text{ratio} = 1$ — метки неотличимы от случайных. Порог блокировки: $\text{ratio} \geq 0.92$.

$k = 5$ — из оригинальных статей (Biggio et al., 2011; Paudice et al., 2019). Paudice et al. показали, что при уровне отравления < 30% KNN-based очистка эффективна; при > 30% чувствительность падает.

**Почему незаменим:** наиболее чувствительный детектор подмены меток без эталонного датасета. Использует прямое противоречие между признаками образца и его назначенной меткой в пространстве признаков.

### 4.12 M10 — Обнаружение почти-дубликатов

*Huang, L. et al. (2011). Adversarial Machine Learning. AISec 2011.*

Для каждого образца находится ближайший сосед. Пара признаётся конфликтующей, если расстояние меньше 5-го перцентиля распределения NN-расстояний **и** метки различаются. Ловит атаку «скопировать пример с минимальными изменениями и подменить метку».

### 4.13 M11 — Isolation Forest

*Liu, F.T., Ting, K.M., Zhou, Z.-H. (2008). Isolation Forest. ICDM 2008.*

*Liu, F.T. et al. (2012). Isolation-Based Anomaly Detection. ACM TKDD, 6(1). DOI: 10.1145/2133360.2133363*

$$s(x, n) = 2^{-E[h(x)]/c(n)}, \quad c(n) = 2H(n-1) - 2(n-1)/n$$

Аномальная точка изолируется за меньшее число разбиений, чем нормальная. Используется `contamination="auto"` (порог на score = −0.5 из оригинальной статьи), что делает долю аномалий данно-зависимой, а не фиксированной. Фиксированный параметр `contamination=0.10` механически помечает ровно 10% точек при `fit_predict`, что лишает детектор информативности.

### 4.14 M12 — Local Outlier Factor

*Breunig, M.M. et al. (2000). LOF: Identifying Density-Based Local Outliers. SIGMOD 2000. (> 7000 цитирований)*

$$\text{LOF}_k(x) = \frac{\sum_{o \in N_k(x)} \text{lrd}_k(o) / \text{lrd}_k(x)}{|N_k(x)|}$$

LOF = 1.0 → нормальная плотность. LOF > 1.5 → аномалия (порог из Breunig et al., 2000). Используются сырые LOF-оценки из `negative_outlier_factor_`, а не бинарные предсказания с фиксированным `contamination` — по той же причине, что и в M11.

**Ключевое отличие от M11:** LOF работает локально и обнаруживает выбросы в неоднородных по плотности распределениях, где Isolation Forest ошибается.

### 4.15 M13 — DBSCAN

*Ester, M. et al. (1996). A Density-Based Algorithm for Discovering Clusters in Large Spatial Databases with Noise. KDD 1996.*

*Schubert, E. et al. (2017). DBSCAN Revisited, Revisited. ACM TODS, 42(3). DOI: 10.1145/3068335*

Шумовые точки (label = −1) — кандидаты в аномалии. $\varepsilon$ выбирается как 90-й перцентиль дистанций до $k$-го соседа (рекомендация Ester et al., уточнённая Schubert et al., 2017).

**Обоснование трёх детекторов плотности (M11–M13):** каждый основан на разном инварианте — глубина изоляции, локальная плотность, кластерная принадлежность. Атака, оптимизированная против одного детектора, неизбежно создаёт детектируемый сигнал для другого. Zhang et al. (2024) [7] показали, что ансамблирование трёх детекторов снижает процент пропущенных отравленных образцов с 12% до 3% по сравнению с любым одиночным.

### 4.16 M14 — Spectral Signatures

*Tran, B., Li, J., Madry, A. (2018). Spectral Signatures in Backdoor Attacks. NeurIPS 2018. arXiv:1811.00636*

Для каждого класса:

1. Центрирование: $\tilde{X}_c = X_c - \bar{X}_c$
2. SVD: $\tilde{X}_c = U \Sigma V^T$, берём $v_1$ — ведущий правый сингулярный вектор
3. Оценка аномальности: $r_i = (x_i^T v_1)^2$
4. Детектирование: $\text{gap} = P_{95}(r) / P_{50}(r) > 20$

**Теорема (Tran et al., 2018):** «все известные backdoor-атаки оставляют спектральный след» — отравленные образцы имеют значительно большую проекцию на $v_1$, чем чистые. Для центрированных гауссовых данных оценки $r_i$ следуют $\chi^2_1$-распределению с $P_{95}/P_{50} \approx 8.4$. Порог 20 выбран выше этого фонового значения.

### 4.17 M15 — Robust PCA

*Candès, E.J., Li, X., Ma, Y., Wright, J. (2011). Robust Principal Component Analysis? JACM, 58(3). DOI: 10.1145/1970392.1970395*

Разложение $X = L + S$, где $L$ — низкоранговая матрица «нормального» поведения, $S$ — разреженные аномалии. В реализации: PCA-реконструкция даёт $L$, ошибка $\|x_i - \hat{x}_i\|_2$ — аппроксимация строки $S_i$. Порог аномальности: $\text{median}(e) + 3.5 \cdot \text{MAD}(e)$ — критерий Iglewicz & Hoaglin.

### 4.18 M16 — Конформный тест сепарабельности

*Sánchez, A. et al. (2025). Provably effective detection of effective data poisoning attacks. arXiv:2501.11795*

Мера несоответствия:

$$\text{score}(x_i) = \frac{d(x_i,\ \text{NN}_{\text{same}}(x_i))}{d(x_i,\ \text{NN}_{\text{diff}}(x_i)) + \varepsilon}, \quad \text{score} > 1 \Rightarrow \text{аномалия}$$

Глобальная статистика $T = \frac{1}{n}\sum_i \mathbf{1}[\text{score}(x_i) > 1]$ сравнивается с пермутационным распределением (300 пермутаций). Уровень значимости $\alpha = 0.10$ повышает чувствительность к умеренным нарушениям.

**Теорема (Sánchez et al., 2025):** если атака отравления достаточно эффективна против произвольного классификатора, датасет обязательно проявит статистически детектируемое нарушение сепарабельности классов.

**Почему незаменим:** единственный метод в блоке 2 с формальной теоретической гарантией. Все остальные — эвристики с настраиваемыми порогами. M16 является финальным арбитром: теорема гарантирует обнаружение любой атаки, реально влияющей на качество модели.

## 5. Блок 3: Анализ обученной модели (Model)

### 5.1 Принцип работы

Блок 3 анализирует обученную модель, используя тот же датасет, на котором она обучалась. Ключевое отличие от блока 2: здесь доступны представления из скрытых слоёв модели, что резко повышает различающую способность детекторов.

Три метода применяются дважды: для табличных MLP (через реконструкцию из `state_dict`) и для fine-tuned BERT (через прямой проход через `transformers`). Итого 6 детекторов + `model_diff` = 7 проверок.

### 5.2 Режимы работы

**Режим 1 (без опорной модели).** Детекторы работают как ранжировщики: помечают верхние `expected_poison` (по умолчанию 15%) строк по каждому скору. Результат — `REVIEW` (мягкое предупреждение) с ранжированным списком подозрительных строк. Без эталона невозможно калиброванно отделить чистые данные от отравленных — только триаж.

**Режим 2 (с опорной моделью и/или чистым сэмплом).** Пороги калибруются по чистым данным через бутстрап:
$$\tau = \text{CI}_{1-\alpha}\left[Q_{1-\alpha}(\text{scores}_{\text{clean}}),\ B=200\right]$$

Строки с $\text{score} > \tau$ — аномалии. При доле превышения ≥ 10% — `BLOCK`, ≥ 3α — `REVIEW`, иначе — `PASSED` (калиброванный вердикт). Это единственный режим, дающий абсолютный вердикт `ALLOW`.

### 5.3 Табличный режим без предварительно обученной модели

Для NLP модель уже существует (fine-tuned BERT). Для табличных данных блок 3 исторически требовал раздельного этапа обучения MLP. Модуль `src/core/tabular_prep.py` устраняет это ограничение: при передаче `--target` CLI автоматически обучает MLP-кандидата (и опорный MLP из `--reference`, если указан) с **общей стандартизацией** — параметры ($\mu$, $\sigma$) фитятся на чистом эталоне и применяются к обоим датасетам.

Это критично для корректности калибровки: при независимой стандартизации двух датасетов активации кандидата и опорной модели оказываются в разных масштабах, что смещает порог $\tau$ и приводит к ложным вердиктам.

### 5.4 SPECTRE (модельный уровень)

*Hayase, J. et al. (2021). SPECTRE: Defending Against Backdoor Attacks Using Robust Statistics. ICML 2021. arXiv:2104.11315*

SPECTRE — улучшение vanilla Spectral Signatures, устраняющее их ключевой изъян: стандартный SVD преследует «мусорную» дисперсию чистых данных (nuisance variance), из-за чего бэкдор-направление не попадает в ведущий сингулярный вектор. SPECTRE убирает эту дисперсию через робастное отбеливание (QUE — quantum entropy):

1. Оценивает ковариацию чистых компонент через MinCovDet или QUE-спектральный оценщик
2. Применяет отбеливающее преобразование $\tilde{R} = \hat{\Sigma}^{-1/2} R$
3. Ищет ведущее направление в отбелённом пространстве

Vanilla Spectral на части backdoor-атак давала AUC 0.21 (хуже случайного), SPECTRE работает стабильно. В системе применяется к активациям последнего скрытого слоя MLP (и `[CLS]`-токенам BERT), а не к входным признакам — это дополнительно усиливает сигнал, поскольку скрытый слой концентрирует семантически значимые различия.

### 5.5 Activation Clustering

*Chen, B. et al. (2018). Detecting Backdoor Attacks on Deep Neural Networks by Activation Clustering. Workshop on Artificial Intelligence Safety, AAAI 2019.*

В пространстве активаций последнего скрытого слоя проводится кластеризация (k-means, k=2) для каждого класса. Бэкдор-отравленные образцы формируют компактный отдельный кластер, хорошо разделённый от чистых (высокий силуэтный коэффициент).

Решение о тревоге принимается при:
- силуэтный коэффициент кластера-меньшинства ≥ 0.45 **и**
- размер кластера-меньшинства ≤ 45% от класса

Пороговая логика дополнена относительным размером кластера для подавления ложных срабатываний на естественно кластеризованных данных.

Для BERT: PCA (10 компонент) → k-means → та же логика.

### 5.6 Randomised Prediction Perturbation (RPP)

*Lin, J. et al. (2026). Detecting Backdoor in Deep Neural Networks via Intentional Adversarial Perturbations.*

Для каждой строки выполняется $n=20$ прогонов с гауссовым шумом $\mathcal{N}(0, 0.3 \cdot \sigma_{\text{col}})$ в пространстве признаков:

$$\text{score}(x_i) = -\frac{1}{n} \sum_{k=1}^{n} \|p(x_i) - p(x_i + \delta_k)\|_\infty$$

Отравленные строки (с вшитым триггером) аномально стабильны: модель предсказывает целевой класс вне зависимости от шума. $L_\infty$-норма используется для максимальной чувствительности к любому изменению предсказания, а не только к изменению вероятности целевого класса.

Для BERT: шум добавляется непосредственно к input embeddings токенов, что эквивалентно пространству входных признаков.

### 5.7 Model Diff

Детектор сравнивает выходные вероятности опорной и кандидатской моделей на каждой строке датасета через Total Variation:

$$\text{TV}(i) = \frac{1}{2}\|p_{\text{new}}(x_i) - p_{\text{ref}}(x_i)\|_1$$

$\text{TV} \approx 0$ → модели согласны (строка, вероятно, чистая). $\text{TV} > 0.5$ → модели кладут большинство массы на разные классы (строка подозрительная). Строки с высоким TV, образующие компактный кластер — признак целевого бэкдора. Широко распределённое умеренное TV — естественный дрейф данных.

**Почему даёт абсолютный вердикт:** опорная модель служит точкой отсчёта, поэтому порог $\text{TV} > 0.5$ семантически осмыслен без дополнительной калибровки.

**Оговорки:** детектор слеп к бэкдорам, присутствующим в обеих моделях; естественный дрейф данных также создаёт расхождение.

## 7. Почему нельзя убрать отдельные методы

**M8 (Mahalanobis)** — единственный метод, учитывающий корреляции между признаками. Стандартный Z-score рассматривает каждый признак независимо и пропускает многомерные выбросы, лежащие в пределах нормы по каждой оси отдельно.

**M9 (KNN)** — наиболее чувствительный детектор подмены меток из всех, не требующих эталона. Paudice et al. (2019) подтвердили эффективность при poison rate < 30%. Без M9 умеренный label flip остаётся необнаруженным блоком 2.

**M11–M13 (IForest, LOF, DBSCAN)** — разные инварианты обнаружения выбросов. IForest нечувствителен к локальной плотности; LOF работает в плотных данных, где IForest ошибается; DBSCAN устойчив к форме кластеров. Исключение любого из трёх снижает покрытие.

**M14 (Spectral, данные)** — теоретически обоснованный детектор бэкдоров на уровне данных. Tran et al. (2018) доказали, что **все** известные backdoor-атаки оставляют спектральный след. Исключение M14 делает систему теоретически небезопасной по отношению к backdoor на уровне данных.

**M16 (Conformal)** — единственный метод с формальной гарантией: любая эффективная атака нарушает сепарабельность. Является финальным арбитром.

**Блок 3 (SPECTRE/AC/RPP на уровне модели)** — незаменим для атак, не оставляющих следов в данных (clean-label attacks [8]), при которых блок 2 бессилен. Lee et al. (2018) [9] показали, что признаки из скрытых слоёв обеспечивают на 15–25% лучшее разделение отравленных образцов по сравнению с входными признаками. SPECTRE дополнительно устойчив к nuisance variance чистых данных, что делает его более надёжным в сценариях с разнородными входными распределениями.

## 8. Ограничения

**Умеренный balanced label flip.** Если атакующий балансирует подмену (равное число 0→1 и 1→0), распределение классов не меняется. M9 и M16 чувствительны к этому, но при небольших объёмах выборки мощности пермутационного теста может не хватить.

**Отсутствие эталонного датасета.** Блок 2 работает без референса: методы M3–M6 могут зафиксировать сдвиг, но не могут однозначно атрибутировать его атаке. PSI (Population Stability Index) [10] закрыл бы эту проблему при наличии чистых данных предыдущего запуска.

**Блок 3 без режима 2.** Без опорной модели детекторы работают как ранжировщики (всегда `REVIEW`), не давая абсолютного вердикта. Это ограничение архитектурное: без калибровки по чистым данным нельзя определить, что является «нормальным» уровнем скоров для данной задачи.

## 9. Заключение

Разработана система обнаружения отравления данных, объединяющая три уровня анализа в единый CLI-пайплайн. Система охватывает 7 классов атак, включая backdoor, label flip, outlier injection, prompt injection, homoglyph substitution и feature perturbation.

Архитектура адаптеров (блок 1: `DatasetGuardScanner`; блок 3: `PostTrainGuardScanner`) обеспечивает независимое развитие монолитных движков без изменения ядра пайплайна. Механизм `@register_scanner` позволяет добавлять новые детекторы без модификации существующего кода.
