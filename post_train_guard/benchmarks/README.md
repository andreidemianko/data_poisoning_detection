# Бенчмарк режима 1 (вариант A + приоритет AC)

Объективная оценка post-train ансамбля **без эталона** (режим 1) на всех датасетах кейса:
чистый `dataset_4` + грязные `dataset_1/2/3`, табличные + NLP (BERT).

## Файлы
| Файл | Что это |
|---|---|
| `bench_mode1.py` | Полный прогон: 6 табличных + 5 NLP × {d4,d1,d2,d3}. Статус методов, вердикт гейта, ROC-AUC. |
| `bench_mode1_nlp_corrected.py` | NLP с **исправленной выборкой** (см. ниже). |
| `validate_modeA.py` | Быстрая проверка табличных решений гейта. |
| `results_mode1_full.csv` | Результаты `bench_mode1.py`. **Табличные строки валидны; NLP-строки — см. оговорку.** |
| `results_mode1_nlp_corrected.csv` | Корректные NLP-результаты. |

## Методология
- **Ground truth (универсально):** строка отравлена, если отличается от позиционно-выровненного
  `dataset_4` (label flip / порча признаков / триггер / гомоглиф — всё видно как расхождение).
- **ROC-AUC** считается там, где яд — подмножество строк (не глобальная трансформация → иначе n/a).
- **Табличные:** полные данные (до 8000 строк), естественная доля яда. MLP обучается под кейс
  (XGBoost нельзя — нет активаций), общая стандартизация.
- **NLP:** полные 47k строк на CPU неподъёмны → подвыборка.

## ⚠️ Оговорка: баг выборки в NLP полного прогона
В `bench_mode1.py` NLP-подвыборка **сохраняла ВСЕ отравленные строки** + добивала чистыми до
лимита → доля яда раздувалась до **50–100%**. Но AC/RPP-гейты предполагают, что яд — это
**меньшинство** (`max_poison_frac=0.45`, `max_frac=0.40`); при яде-большинстве они структурно
не срабатывают → NLP-recall в `results_mode1_full.csv` **занижен** (артефакт, не свойство метода).

**Исправление** (`bench_mode1_nlp_corrected.py`): документированные атаки bitext (backdoor-токен /
гомоглиф-кириллица, детектим напрямую) инжектированы как **меньшинство 15%**; остальное —
случайная выборка. Используйте этот файл для NLP-выводов.

## Ключевые цифры (режим 1)
**Ранжирование (сильная сторона) — ROC-AUC:**
- таблицы, порча признаков: uci d2 AC **1.0**, creditcard d3 AC 0.99, uci d3 0.96, taiwan d3 0.95
- текст/BERT: homoglyph SPECTRE **0.90** / AC 0.85; backdoor AC **0.76**

**Вердикт гейта:**
- Чистое → ALLOW: **8/11** (таблицы 5/6, NLP 3/5). Все FP — на данных с очень тесной природной
  структурой классов (bank_churn; bitext/banking77 — десятки интент-категорий).
- Документированные NLP-атаки (15% меньшинство): **2/2 → REVIEW** (backdoor, homoglyph).
- label_flip → слеп для всех трёх (зона model_diff / kNN).

**Вывод:** без эталона model-level — сильный **ранжировщик** (триаж), но не надёжный
**классификатор** «чисто/грязно» (rank-vs-gate gap). Абсолютный вердикт (вкл. BLOCK) — режим 2.

## Запуск

```bash
source .venv/bin/activate
KMP_DUPLICATE_LIB_OK=TRUE python3 post_train_guard/benchmarks/bench_mode1.py            # таблицы + NLP

```
